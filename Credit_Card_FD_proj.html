Credit Card Default Detection

This study predicts credit card defaults using mathematical and statistical tools in Python, analyzing a large dataset to support financial risk assessment.

Code on GitHub Full Report

Background

Credit card defaults pose significant risks to financial institutions, necessitating accurate predictive models. This project utilizes a dataset from platforms like Kaggle, containing 307,511 records with 122 features, to explore repayment behavior. Conducted as part of the MA-308 Mini Project at Sardar Vallabhbhai National Institute of Technology, Surat, the study was led by Ayushi Singh, Kaushik Sanghani, and Suryam Gupta under Dr. Jayesh M. Dhodiya’s supervision, completed in May 2022.

Research Question

How can features such as payment history, employment duration, credit amount, and demographic data be used to predict credit card defaults using logistic regression and statistical analysis?

Hypothesis

“Features like payment history, employment duration, credit amount, and demographic data will exhibit varying correlations with credit card defaults, with logistic regression identifying key predictors and thresholds for risk assessment.”

This is informed by data mining principles and financial risk modeling research.

Dataset

The dataset, sourced from platforms like Kaggle, includes 307,511 rows and 122 columns, covering client data such as payment history, employment duration, credit amounts, and demographics. Before cleaning, it occupied 286.2 MB, with raw figures reflecting a diverse set of financial and personal attributes.

Data Pre-processing and Cleaning

The dataset was loaded into Python dataframes, reducing memory usage from 286.2 MB to 92.38 MB (67.7% reduction) by optimizing data types. Missing data was handled by replacing with mean/mode values or interpolation, outliers (e.g., 'DAYS_EMPLOYED' value of 365,243) were corrected with the mean, and categorical variables (e.g., 'CODE_GENDER') were label-encoded. Post-cleaning, the dataset retained 307,511 rows with relevant features.

Exploratory Data Analysis (EDA)

EDA revealed a target distribution with 91.93% non-defaults and 8.07% defaults (imbalance ratio: 11.39). Heatmaps identified correlations, e.g., AMT_CREDIT and AMT_GOODS_PRICE showed high positive correlation, aiding feature selection for modeling.

Model and Findings

The project used logistic regression to predict defaults (1) vs. non-defaults (0), with key findings:

  1. Payment History: Highly correlated with defaults, with missed payments indicating higher risk.
  2. Employment Duration: Longer employment (adjusted for outliers) negatively correlates with defaults.
  3. Credit Amount (AMT_CREDIT): Higher amounts correlate with increased default risk when paired with low repayment capacity.
  4. Demographic Data: Features like gender and income bracket showed moderate influence, with verified data enhancing model reliability.

Model accuracy was 91.92%, but overfitting occurred due to data imbalance, with 56,536 correct non-default predictions out of 61,502 test cases.

Conclusion

The logistic regression model achieved 91.92% accuracy in predicting credit card defaults, driven by features like payment history, employment duration, and credit amount. However, the model overfit due to a 91.93% non-default majority, limiting its ability to predict defaults (0% correct). Future improvements include undersampling, cross-validation, and metrics like Precision and Recall to address imbalance. The study highlights the importance of balanced datasets and robust modeling for effective financial risk assessment.
